# 문서 플랫폼 아키텍처 (StoryBakery)

## 배경
조직 내 여러 레포가 문서를 운영하면서도, 시각적 경험과 정보 구조는 통일되고
프로젝트별 커스텀은 크게 허용되는 구조가 필요합니다.

## 목표
- 조직 공통 템플릿을 통해 문서 경험을 통일한다.
- Luau/Moonwave 기반과 비-Luau 프로젝트를 모두 지원한다.
- 애니메이션과 인터랙션이 풍부한 테마를 유지한다.
- 프로젝트별 커스텀성을 크게 보장한다.

## 비목표
- 모든 프로젝트를 단일 문서 사이트로 강제 통합하지 않는다.
- 특정 배포 환경에 종속된 설계를 강요하지 않는다.

## 설계 요약
- 기본 전략은 공통 테마/프리셋을 패키지로 배포하고 프로젝트는 얇은 설정과 콘텐츠만 유지한다.
- Docusaurus를 공통 뼈대로 사용한다.
- Moonwave 출력물은 생성물로 취급하고, Docusaurus에 연결한다.
- 템플릿 레포는 문서 사이트 부트스트랩 용도로만 사용한다.

## 구성 요소
- 공통 테마 패키지: 디자인 토큰, 컴포넌트, 애니메이션
- 공통 프리셋: 플러그인 조합과 기본 옵션
- Moonwave 어댑터: 출력 경로를 문서 소스로 연결
- 템플릿 레포(선택): Docusaurus 사이트 스캐폴딩

## 데이터/빌드 흐름
```
프로젝트 문서 소스
  - (선택) Moonwave 출력 -> docs/generated
  - 수동 문서 -> docs/manual
  - 커스텀 페이지 -> src/pages
            |
            v
     Docusaurus 빌드
            |
            v
        정적 사이트
```

## 커스텀성 전략
- 디자인 토큰과 CSS 변수를 통해 전역 스타일을 제어한다.
- 공통 컴포넌트로 레이아웃과 모션을 제공한다.
- 프로젝트는 필요한 범위에서 컴포넌트를 확장한다.

## 운영 및 업데이트
- 공통 테마/프리셋은 버전으로 관리하고 프로젝트에서 고정한다.
- 템플릿 레포는 부트스트랩 용도로만 최소 유지한다.
- 문서 콘텐츠는 프로젝트 레포가 소유한다.
