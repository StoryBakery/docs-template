# 기여 가이드

기여는 이 저장소의 표준 구조와 문서 정책을 따라 진행합니다.

## 기본 원칙
- 생성물(`.generated/`, `docs/reference/`)은 수정하지 않습니다.
- 문서 변경 시 `website/docs/`와 사이드바를 함께 갱신합니다.
- 로컬 빌드/테스트 후 PR을 올립니다.

## 브랜치와 커밋
- 기능 단위 브랜치에서 작업합니다.
- 변경 범위가 명확한 커밋 메시지를 사용합니다.

## 테스트
```
npm run test
```

## 문서 변경
- `website/docs/`에 문서를 추가/수정합니다.
- `website/sidebars.ts`에 노출 위치를 반영합니다.
